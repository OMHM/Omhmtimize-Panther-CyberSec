Bad USB Basics
*Note* More Content Coming Soon Key Concepts: What's a Rubber Ducky?

  1. A Bad Usb is a term used as a blanket term for almost any Malicious USB device. The limitations are pretty endless but require you to have physical access to the device. The Rubber Ducky Bad Usb is one of several methods or ways to create and use the Bad USB attack vectors for security testing purposes.
  2. The Rubber Ducky Bad Usb is a fairly simplistic due to its hardware and user-friendly programming syntax and semantics. Basically it is fairly easy for a human to read and deduce what the code is doing based of the words being used. With the vast number of prewritten scripts, an online encoder, and decoder hosted at ducktoolkit.com , the Rubber Ducky Usb is a great kick off point in learning about Security of Corporate Infrastracture as well as foundational concepts of Computational Thinking and Computer Science.
Hardware: Three Key Elements
  1. The USB Keyboard Emulator with onboard SDHC Micro Sd Card Reader. This hardware device contains two major componets for the implementation of the hack. First the physical hardware components make up the USB jack and the memory card reader. These two components provide the physical connection for the attack. The firmware is the onboard-software and is the code responsible for emulating/faking the firmware of a keyboard, establishing a connection to the PC, allowing the Bad Usb to execute keyboard instructions, and maintaining the connection
  2. The Encoder or Encoding Device that will write your code onto the Micro SD Card
  3. Lastly you have the Micro Sd Card itself. This contains the actual code/script that is ran by the victims computer when the bad usb is connected.

# MISSION
Omhmtimize strives to create a streamlined learning experience for modern society. The difficulty of learning a new subject can be a huge deterent. Compounding the frustration of learning newsubjects, the overwhelming lack of valid or sufficient resources form a hostile enviroment. Here at OMHM, we are constantly developing a model of human learning in an effort to make learning less frightnening and possible for the global community. By vigorously striving to source only the most relavant and accurate data as well as continually working to perfect the academic paradigm, we hope to one day leave no corner of the globe in the dark.
Omhmtimize strives to:
  1. Create an open source communal learning enviroment where all can come to seek knowledge
  2. Create a platform for intellectuals to share expertise and experience globally in an effort to educate the unfortunates unable recieve adaquete education.
  3. Document and classify learning methods in an effort to implement the most efficient lesson style for each individual learner.
  4. Create a paradigm that can be used to automate the learning process
